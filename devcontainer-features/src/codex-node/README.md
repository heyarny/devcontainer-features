# OpenAI Codex with Node.js Dev Container Feature

Installs Node.js, npm, the OpenAI Codex CLI, and can link Codex state folders
from `$CODEX_HOME` into the workspace after the workspace mount is available.

Consumers do not need to add a separate Node/npm feature. This feature installs
Node.js 24 with nvm and npm 11.15.0 by default, and can
override either version when needed.

## Options

| Option | Default | Description |
| --- | --- | --- |
| `codexVersion` | `latest` | Version or npm dist-tag of `@openai/codex` to install. |
| `nodeVersion` | `24` | Node.js version or nvm alias to install. |
| `npmVersion` | `11.15.0` | npm version or dist-tag to install. Use `bundled` or `none` to keep the npm version included with Node.js. |
| `linkedFolders` | empty | Optional folder mappings. Leave empty to disable linking. |

Dev Container Feature options are officially strings or booleans, but the
current Dev Container CLI accepts an array of strings in `devcontainer.json` and
passes it to the feature as a comma-separated value. That gives readable
one-mapping-per-line configuration while keeping the feature option compatible
with the spec.

Each `linkedFolders` entry uses `name=target`. The `name` is created under
`$CODEX_HOME`. Relative `target` values resolve under the workspace.
Explicit paths may use `{workspace}`, `{workspaceFolder}`, or
`{localWorkspaceFolder}` placeholders.

Prefer relative paths or brace placeholders. `$workspace`-style values are
interpreted by the feature install shell unless escaped before they reach this
feature.

## Example

Install Codex only:

```jsonc
{
  "features": {
    "ghcr.io/heyarny/devcontainer-features/codex-node:1": {}
  }
}
```

Install Codex and link workspace-backed state folders:

```jsonc
{
  "features": {
    "ghcr.io/heyarny/devcontainer-features/codex-node:1": {
      "codexVersion": "latest",
      "linkedFolders": [
        "sessions=.codex/sessions",
        "archived_sessions=.codex/archived_sessions"
      ]
    }
  }
}
```

The fully spec-compatible single-string form for the same links is:

```jsonc
{
  "features": {
    "ghcr.io/heyarny/devcontainer-features/codex-node:1": {
      "linkedFolders": "sessions=.codex/sessions,archived_sessions=.codex/archived_sessions"
    }
  }
}
```

JSON-encoded strings are also accepted when generated by tooling, but arrays of
objects are not portable with the current Dev Container CLI.

Explicitly disable workspace state links:

```jsonc
{
  "features": {
    "ghcr.io/heyarny/devcontainer-features/codex-node:1": {
      "linkedFolders": ""
    }
  }
}
```
